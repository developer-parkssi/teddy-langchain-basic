## Chapter01 RAG을 사용해야 하는 이유

- GPT가 출시되고, 여러 고급 기능이 보완되면서 사용자들은 단순히 질문받고 답변받는 것에서 더 나아가 전문적인 영역까지 적극적으로 활용하기 시작
- 그런데 영역이 확대됨에 따라 문제점들 발생 시작

1. ChatGPT의 지식 컷오프
   - 최신 정보에 대한 질문에 대해 부정확한 답변을 제공할 수 있음
2. 개인이나 회사의 내부 데이터가 학습되어 있지 않아, 특정 도메인에 대한 질문에는 기대하는 답변 X
3. ChatGPT에 개인이나 회사 정보를 담은 문서를 업로드하면 보안상 문제 발생
4. 문서의 양이 많아질수록 할루시네이션 현상 발생

### ChatGPT의 한계를 보완하기 위해 주목받은 RAG

- **RAG**이란 **Retrieval-Augmented Generation**의 약자로, 검색(Retrieval), 증강(Augmented), 생성(Generation)이라는 의미
- **거대 언어 모델**(LLM; Large Language Model)이 외부의 신뢰할 수 있는 지식 데이터베이스를 참조하여 최적화된 응답을 생성하는 기술
- 적절한 RAG을 적용하면 앞서 언급한 문제점들을 다음과 같이 개선

1. 최신 정보를 기반으로 답변 할 수 있으며, LLM이 정보를 찾을 수 없는 경우 '검색' 기능을 활용해 답변 제공 가능
2. 회사 내부에 데이터베이스를 구현함으로써 개인이나 회사의 내부 데이터를 참고하여 GPT가 답변 가능
3. 문서를 내부 데이터 베이스에 저장하고 지속적으로 데이터 축적 가능, 저장된 데이터베이스에서 원하는 정보를 검색한 후 이를 바탕으로 답변 생성 가능
4. 저장된 데이터베이스에서 답변의 출처를 역으로 검색하고 검증하는 방식으로 할루시네이션 현상 감소

- RAG을 사용하면 GPT가 새롭게 축적되는 데이터베이스 기반으로 답변하여 사용자는 데이터베이스만 업데이트하면 더 높은 품질의 최신 답변을 얻을 수 있다
- RAG은 사전 학습된 지식에만 의존하지 않고 참고할 만한 자료를 주면 그것을 토대로 답변하여 훨씬 더 정확
- 쉽게말해 사전 학습된 GPT에 정보 검색 기능을 추가한 것. 검색 대상은 문서와 인터넷은 물론 다양한 도메인일 수도 있다
- 우리가 할 일은 다양한 정보 검색 기술을 활용하여 RAG 시스템을 점점 고도화 하는 것    
    Ex) 모든 문서를 GPT에 제공하는 대신 핵심 정보만 요약해서 전달하면 훨씬 더 효율적으로 참고
- GPT가 참고할 데이터를 잘 정제하면 RAG 시스템 성능 향상

### 커스터마이즈가 어려운 ChatGPT 내장 RAG 시스템

- ChatGPT에 문서를 업로드하여 상세 질문을 하면 대답을 못할 때가 있음
- ChatGPt에서 RAG가 작동하는 과정은 OpenAI에서 공개하고 있지 않아 사용자가 일르 직접 제어 X
- 문서를 ChatGPT가 최대한 잘 검색할 수 있는 형태로 변경하는 것이 현재로서는 최선의 방법    
  Ex) 자주하는 질문을 FAQ로 만들거나 PDF, DOCX, HWP 등의 문서를 마크다운 형식으로 작성하는 방식
- 하지만 모든 문서를 이렇게 변경하는 것은 많은 시간과 노력이 필요한 작업
- 이러한 한계 극복 위해 LangChain을 이용해 RAG 시스템을 직접 구축해볼 것
