# Chapter13 리트리버

- 지정된 벡터 데이터베이스에서 사용자 질문이 들어오면 그 질문으로 우리가 저장한 문서에서 검색하는 과정

## 리트리버 작동 방식

- 사용자의 질문을 처리하기 위해 먼저 질문을 벡터 형태로 변환
- 저장된 문서 벡터들과 질문 벡터 간의 유사도를 계산
- 이를 위해 코사인 유사성이나 MMR (Maximal Marginal Relevance)와 같은 방법을 사용
- 유사도가 높은 문서들을 선택하여 사용자에게 반환

## 희소 리트리버와 밀집 리트리버

### 희소 리트리버

- TF-IDF (Term Frequency-Inverse Document Frequency)와 같은 방법을 사용하여 문서와 질문을 벡터로 표현
  - TF-IDF
    - 단어의 빈도와 역문서 빈도를 고려하여 단어의 중요도를 계산하는 방법
    - 자주 나타나면서도 문서 집합 전체에서 드물게 나타나는 단어가 높은 가중치
  - BM25
    - TF-IDF의 개선된 버전으로, 문서 길이와 단어 빈도를 고려하여 가중치를 계산하는 방법
    - 긴 문서가 짧은 문서보다 불리하지 않도록 조정
- 각 단어의 존재 여부만을 고려하기 때문에 계산 비용이 낮고 구현이 간단

### 밀집 리트리버

- 최신 딥러닝 기술을 사용하여 문서와 쿼리를 연속적인 고차원 벡터로 인코딩
- 벡터 공간에서의 거리를 사용하여 질문과 가장 관련성 높은 문서를 찾는다
- 언어의 뉘앙스와 문맥을 이해하는 데 유리하며, 복잡한 질문에 대해 더 정확한 검색 결과 제공
